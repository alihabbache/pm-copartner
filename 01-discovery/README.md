# 01 — Insight: User Research Synthesis

## What It Does

Takes your raw user interview notes — whether that's a folder of docs, a Notion page, bullet points in an email, or a paste of messy text — and transforms them into a **structured one-page insight report** with:

- **Key themes** surfaced across interviews
- **Supporting quotes** for each theme
- **Ranked opportunity areas** by frequency and impact

## Why This Exists

Most PMs have the interviews. Few have the time to synthesise them before the next planning meeting. This agent does the synthesis pass so you walk in knowing your users.

## When to Run

- After every research sprint (even just 3–5 interviews)
- Before quarterly planning
- Any time you're about to make a prioritisation decision and want evidence

## Output Format

```
# Research Synthesis — [Date / Sprint]

## Top Themes
1. [Theme] — X mentions
   - Supporting quote: "..."
   - Supporting quote: "..."

2. [Theme] — X mentions
   ...

## Ranked Opportunity Areas
| Opportunity | Frequency | Severity | Notes |
|-------------|-----------|----------|-------|
| ...         | High      | High     | ...   |

## Next Steps
- ...
```

## Minimum Requirements

- Your interview notes (any format: bullet points, transcript, doc)
- Nothing else required

## With Integrations

- Notes stored in **Notion** → use Notion MCP to pull them automatically
- Save output to **Google Drive** → use Drive MCP to write the report
- Track opportunities in **Linear / Jira** → manually copy ranked items or use connector
